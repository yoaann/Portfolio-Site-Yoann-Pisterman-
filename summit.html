Money saving app Summit

Overview

Summit, a money saving app that rises to the bright side of the banking experience. Designing a brand identity that resonates with non-tech users on a human level & developping a smooth user flow experience with useful features.

This app is a tool that helps to balance your saving goals by priorities can help managing your finance at a glance.

User flow

Defining the right process rhythm through the user flow.

A smooth user flow process that links the user bank account and creates a first saving goal, without any frustration or discouraging feelings

01

As a new user, I need to be able to create a profile, so that the financial advice I receive is personalized to me, my spending habits, and tailored to my goals.

02

As a money-saver, I need to be able to input information on the money I am receiving and spending (and on what), so that I can see an overview of my finances.

03

As a user, I want to see a dashboard of my finances clearly and visually, so that I can see how much I am spending on what at a glance.

Create a goal screens

There is a lot of reasons why to save money. There is list of common goals, but the user has also the possibility to search for more suited choices.

If the user inputs the recurrence it will calculate the duration, in another word: how long it’s going to take to get to your goal.

Inputting the duration would influence the recurrence; so that saving amount is calculated automatically from the user’s spending habits.

Goal screen

The user can adapt the goals at anytime and is also able to change their trajectory if needed.

In order to deliver a tailored goal saving strategy plan, this extra feature helps and encourages the user through the saving money process.

Reflection

It was a challenging project because it was the first time I approached making a money saving app, and became a very instructive project on the UI and UX perspective.

Understanding the problem of the design and finding solutions were the most challenging part of the process. There are many ways to achieve a great design experience, and I had to analyse many other existing apps, in order to understand what problem they were trying to solve.

After gathering enough information, design ideas, and mid-fidelity wireframing; the user testing phase allowed me to bring the design into a realistic user experience.
